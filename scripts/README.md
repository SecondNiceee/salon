# Scripts

Набор скриптов для управления данными проекта через REST API.

## Требования

- Node.js 18+ / 20+
- pnpm
- Запущенный сервер (по умолчанию `http://localhost:3000`)
- Переменные окружения в `.env`:
  - `NEXT_PUBLIC_SERVER_URL` - URL сервера (опционально, по умолчанию `http://localhost:3000`)
  - `PAYLOAD_API_KEY` - API ключ для авторизации (обязательно для записи/удаления)

---

## Доступные скрипты

### Просмотр категорий

```bash
pnpm list:categories
```

**Что делает:**
- Выводит список всех категорий и подкатегорий из базы данных
- Показывает `id`, `value`, `title` и вложенную структуру
- Полезно для проверки структуры категорий перед сидированием фильтров

**Пример вывода:**
```
Категории (всего: 6):
├── [1] kursy-massage - Курсы массажа
│   ├── [10] classic-massage - Классический массаж
│   └── [11] sport-massage - Спортивный массаж
├── [2] kursy-kosmetologa - Курсы косметологии
...
```

### Создание/обновление FilterConfig

```bash
pnpm seed:filters
```

**Что делает:**
- Создаёт или обновляет FilterConfig для 6 категорий:
  - `kursy-massage` - Курсы массажа
  - `kursy-kosmetologa` - Курсы косметологии
  - `kursy-tattoo` - Курсы тату
  - `spa-main` - СПА
  - `massage-main` - Массаж
  - `gifts` - Подарочные сертификаты
- Для каждой категории ищет ID через REST API
- Если FilterConfig уже существует - обновляет (PATCH)
- Если не существует - создаёт (POST)

**Фильтры включают:**
- Основные фильтры (`isAdvanced: false`) - видны сразу
- Расширенные фильтры (`isAdvanced: true`) - скрыты под кнопкой "Ещё параметры"
- Типы: `radio` (один вариант), `checkbox` (несколько вариантов)

## Типичный workflow

1. **Проверить категории:**
   ```bash
   pnpm list:categories
   ```

2. **Очистить старые фильтры (опционально):**
   ```bash
   pnpm delete:filters
   ```

3. **Создать фильтры:**
   ```bash
   pnpm seed:filters
   ```

4. **Проверить результат в админке:**
   - Открыть `/admin/collections/filter-configs`

---

## Структура FilterConfig

Каждый FilterConfig привязан к категории и содержит массив фильтров:

```typescript
interface FilterConfig {
  category: number            // ID категории
  filters: {
    key: string               // Уникальный ключ (goal, direction, document...)
    label: string             // Отображаемое название
    type: "checkbox" | "radio"
    isAdvanced: boolean       // true = скрыт под "Ещё параметры"
    options: {
      value: string           // Значение для фильтрации
      label: string           // Отображаемый текст
    }[]
  }[]
}
```

---

## Настройка фильтров

Все фильтры определены в файле `scripts/seed-filter-configs.ts` в массиве `filterConfigs`.

Для добавления/изменения фильтров:
1. Отредактируйте массив `filterConfigs`
2. Запустите `pnpm seed:filters` (обновит существующие)

## Troubleshooting

**Ошибка 403 Forbidden:**
- Убедитесь, что `PAYLOAD_API_KEY` установлен в `.env`
- Проверьте, что ключ валидный

**Категория не найдена:**
- Запустите `pnpm list:categories` для проверки существующих категорий
- Убедитесь, что `value` категории совпадает с `categoryValue` в скрипте

**Сервер недоступен:**
- Убедитесь, что сервер запущен (`pnpm dev`)
- Проверьте `NEXT_PUBLIC_SERVER_URL` в `.env`
