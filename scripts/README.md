# Scripts

Набор скриптов для управления данными проекта через **Payload Local API**.

> Скрипты используют `overrideAccess: true`, что позволяет обходить проверки доступа (не нужен API ключ или права админа).

## Требования

- Node.js 18+ / 20+
- pnpm
- Переменные окружения в `.env`:
  - `DATABASE_URI` - строка подключения к PostgreSQL (обязательно)
  - `PAYLOAD_SECRET` - секрет Payload (обязательно)

> **Важно:** Сервер НЕ должен быть запущен для работы скриптов. Скрипты подключаются напрямую к БД через Payload Local API.

---

## Доступные скрипты

| Команда | Описание |
|---------|----------|
| `pnpm list:categories` | Показать все категории и подкатегории |
| `pnpm seed:filters` | Создать/обновить FilterConfig для всех категорий |
| `pnpm delete:filters` | Удалить ВСЕ FilterConfig |
| `pnpm seed:cities` | Сидирование городов и регионов |

### 2. Создание/обновление FilterConfig

```bash
pnpm seed:filters
```

**Что делает:**
- Создаёт или обновляет FilterConfig для 6 категорий:

| categoryValue | Название |
|---------------|----------|
| `kursy-massage` | Курсы массажа |
| `kursy-kosmetologa` | Курсы косметологии |
| `kursy-tattoo` | Курсы тату |
| `spa-main` | СПА |
| `massage-main` | Массаж |
| `gifts` | Подарочные сертификаты |

- Для каждой категории ищет ID в БД
- Если FilterConfig уже существует - обновляет
- Если не существует - создаёт новый

**Фильтры включают:**
- **Основные** (`isAdvanced: false`) - видны сразу
- **Расширенные** (`isAdvanced: true`) - скрыты под кнопкой "Ещё параметры"
- **Типы:** `radio` (один вариант), `checkbox` (несколько вариантов)

## Типичный workflow

```bash
# 1. Проверить категории
pnpm list:categories

# 2. Очистить старые фильтры (опционально)
pnpm delete:filters

# 3. Создать фильтры
pnpm seed:filters

# 4. Проверить результат в админке
# Открыть /admin/collections/filter-configs
```

---

## Структура FilterConfig

Каждый FilterConfig привязан к категории и содержит массив фильтров:

```typescript
interface FilterConfig {
  category: number            // ID категории
  filters: {
    key: string               // Уникальный ключ (goal, direction, document...)
    label: string             // Отображаемое название
    type: "checkbox" | "radio"
    isAdvanced: boolean       // true = скрыт под "Ещё параметры"
    options: {
      value: string           // Значение для фильтрации
      label: string           // Отображаемый текст
    }[]
  }[]
}
```

---

## Настройка фильтров

Все фильтры определены в файле `scripts/seed-filter-configs.ts` в массиве `filterConfigs`.

**Для добавления/изменения фильтров:**
1. Отредактируйте массив `filterConfigs`
2. Запустите `pnpm seed:filters` (обновит существующие)

## Troubleshooting

### Ошибка подключения к БД
- Убедитесь, что `DATABASE_URI` установлен в `.env`
- Проверьте, что PostgreSQL доступен

### Категория не найдена
- Запустите `pnpm list:categories` для проверки существующих категорий
- Убедитесь, что `value` категории совпадает с `categoryValue` в скрипте

### Ошибка "Cannot find module"
- Убедитесь, что зависимости установлены: `pnpm install`
- Проверьте, что TypeScript компилируется без ошибок
